# Agent Instructions

## After Each Milestone
1. Run all available tests
2. Commit the changes with a descriptive commit message after the tests pass
3. Push the commit after the tests pass
4. Mark the completed todo(s) as done in the plan
5. Update all relevant documentation to reflect the current state of the project

## Documentation Standards
- The project must be documented at all times, including the plan, architecture, and any decisions made
- When a task is completed, check it off in the plan
- When all tasks in a milestone are complete, audit ALL project documentation to ensure it accurately reflects the current state:
  - `README.md` — setup instructions, scripts, and feature list must stay current
  - `agents.md` — these instructions (update if the workflow changes)
  - `plan.md` (in session state) — completed todos marked done, notes updated
  - Inline code comments — remove outdated comments, add comments for non-obvious logic
- Never leave stale documentation (e.g. references to files that no longer exist, outdated instructions, or features that changed)
